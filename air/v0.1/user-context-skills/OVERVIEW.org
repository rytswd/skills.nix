* Summary

Ship a small *suite* of opinionated-but-empty skill templates that
capture global, user-scoped context for agents (pi / codex / claude /
gemini) running on the user's machine. Each skill in the suite has a
distinct trigger condition, matching the documented best-practice
guidance from Anthropic and OpenAI.

This is the index document. Each significant decision lives in its
own focused doc.

* Goals (recap)

- Suite of small, focused templates — *not* one omnibus file.
- Each template has a tight =description= with explicit trigger
  phrases so agents implicitly invoke the right one at the right time.
- Templates are mostly empty (section headers + inline prompts); no
  prescriptive content. =skills.nix= does not opine on =nix run= vs
  =nix shell=, editor choice, etc.
- Two wiring options on top of the existing =localSkills= and =skills=
  surfaces. No new module options.

* Non-goals

- Prescribing what users put in each file.
- Autofill / machine detection.
- Multi-host selection (one template per host) — users can layer that
  in their own home-manager config.
- Replacing =AGENTS.md=. These skills carry *global* preferences;
  project-specific guidance stays in =AGENTS.md=.
